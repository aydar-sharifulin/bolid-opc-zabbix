# Bolid OPC UA → Zabbix

Мониторинг объектов системы **БОЛИД Орион Про** через OPC UA с передачей данных в Zabbix.

Проект предназначен для автоматического обнаружения объектов системы БОЛИД через OPC UA, чтения их текущего состояния и передачи информации в Zabbix для дальнейшего мониторинга и визуализации.

## Возможности

- автоматическое обнаружение объектов OPC UA;
- формирование Low-Level Discovery (LLD) для Zabbix;
- чтение текущих состояний объектов;
- расшифровка кодов состояний БОЛИД;
- передача данных через `zabbix_sender`;
- отслеживание изменений состояний между циклами опроса.

## Поддерживаемые типы объектов

- `Input_`
- `Output_`
- `Section_`
- `GroupSection_`
- `Reader_`
- `Door_`
- `Camera_`
- `Device_`

## Требования

- Python 3.9 или новее;
- доступ к OPC UA-серверу **БОЛИД Орион Про**;
- установленный `zabbix_sender`;
- Zabbix Server или Zabbix Proxy;
- библиотека Python `opcua`.

## Установка

## Основные компоненты

### `bolid/discover.py`

Подключается к OPC UA-серверу, рекурсивно обходит дерево объектов и формирует Low-Level Discovery JSON для Zabbix.

Ключ обнаружения:

```text
bolid.discovery
```

---

### `bolid/poll.py`

Читает текущие состояния обнаруженных объектов, расшифровывает коды состояний БОЛИД и передает информацию в Zabbix.

Используемые ключи:

```text
bolid.state_code[{#TYPE},{#ID}]
bolid.state_text[{#TYPE},{#ID}]
bolid.name[{#TYPE},{#ID}]
bolid.path[{#TYPE},{#ID}]
```

---

### `bolid/state_map.py`

Содержит словари соответствия кодов состояний БОЛИД их текстовому представлению.

Используется скриптом опроса для преобразования числовых кодов в понятные описания.

## Настройка

Перед запуском необходимо указать параметры подключения:

```text
OPC_URL
ZABBIX_SERVER
ZABBIX_HOST
STATE_FILE
```

Пример:

```text
OPC_URL=opc.tcp://192.168.10.100:4840
ZABBIX_SERVER=127.0.0.1
ZABBIX_HOST=Bolid OPC
STATE_FILE=/var/lib/bolid-opc-zabbix/last_states.json
```

> **Важно:** не размещайте реальные адреса серверов, учетные данные и внутреннюю инфраструктурную информацию в публичном репозитории.

## Запуск обнаружения

Из корня проекта выполните:

```bash
python3 -m bolid.discover
```

## Запуск опроса

Из корня проекта выполните:

```bash
python3 -m bolid.poll
```

## Принцип работы

1. Скрипт подключается к OPC UA-серверу.
2. Находит корневой узел системы БОЛИД.
3. Рекурсивно обходит дерево объектов.
4. Находит поддерживаемые типы объектов.
5. Получает переменные `Name` и `State`.
6. Скрипт обнаружения формирует и отправляет Low-Level Discovery в Zabbix.
7. Скрипт опроса считывает код состояния объекта.
8. Код состояния преобразуется в текстовое описание.
9. Значения передаются в Zabbix через `zabbix_sender`.
10. Предыдущее состояние сохраняется локально для регистрации изменений.
