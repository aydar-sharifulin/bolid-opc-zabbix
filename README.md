# Bolid OPC UA → Zabbix

Мониторинг объектов системы **БОЛИД Орион Про** через OPC UA с передачей данных в Zabbix.

Проект выполняет:

* автоматическое обнаружение объектов OPC UA;
* формирование Low-Level Discovery для Zabbix;
* чтение текущих состояний объектов;
* расшифровку кодов состояний БОЛИД;
* передачу значений через `zabbix_sender`;
* отслеживание изменений состояний между опросами.

## Поддерживаемые типы объектов

* `Input_`
* `Output_`
* `Section_`
* `GroupSection_`
* `Reader_`
* `Door_`
* `Camera_`
* `Device_`

## Требования

* Python 3.9 или новее;
* доступ к OPC UA-серверу Орион Про;
* установленный `zabbix_sender`;
* Zabbix Server или Zabbix Proxy;
* Python-библиотека `opcua`.

## Установка

Клонируйте репозиторий:

```bash
git clone https://github.com/USERNAME/bolid-opc-zabbix.git
cd bolid-opc-zabbix
```

Создайте виртуальное окружение:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

Установите зависимости:

```bash
pip install -r requirements.txt
```

## Основные компоненты

### `discover.py`

Подключается к OPC UA-серверу, обходит дерево объектов и формирует JSON для Low-Level Discovery Zabbix.

Ключ обнаружения:

```text
bolid.discovery
```

### `poll.py`

Читает состояния обнаруженных объектов, расшифровывает коды и передаёт данные в Zabbix.

Используемые ключи:

```text
bolid.state_code[{#TYPE},{#ID}]
bolid.state_text[{#TYPE},{#ID}]
bolid.name[{#TYPE},{#ID}]
bolid.path[{#TYPE},{#ID}]
```

## Настройка

Перед запуском необходимо указать:

```text
OPC_URL
ZABBIX_SERVER
ZABBIX_HOST
STATE_FILE
```

Пример параметров:

```text
OPC_URL=opc.tcp://192.168.10.100:4848
ZABBIX_SERVER=127.0.0.1
ZABBIX_HOST=Test OPC
STATE_FILE=/var/lib/bolid-opc-zabbix/last_states.json
```

Не размещайте реальные адреса, учётные данные и внутренние имена узлов в публичном репозитории.

## Запуск обнаружения

```bash
python3 discover.py
```

## Запуск опроса

```bash
python3 poll.py
```

## Принцип работы

1. Скрипт подключается к OPC UA-серверу.
2. Находит рабочий узел Орион Про.
3. Рекурсивно обходит дочерние объекты.
4. Для каждого поддерживаемого объекта находит переменные `Name` и `State`.
5. Скрипт обнаружения отправляет JSON в Zabbix.
6. Скрипт опроса читает код состояния.
7. Код преобразуется в текстовое описание.
8. Значения передаются в Zabbix через `zabbix_sender`.
9. Предыдущее состояние сохраняется локально для регистрации изменений.

## Безопасность

Рекомендуется:

* ограничить сетевой доступ к OPC UA-серверу;
* запускать скрипты от отдельного системного пользователя;
* не хранить пароли и адреса в исходном коде;
* использовать переменные окружения или отдельный конфигурационный файл;
* исключить файл состояний и локальные журналы из Git.
